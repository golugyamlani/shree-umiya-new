This skill guides the creation of distinctive, production-grade websites and frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

**RULE OF PRECEDENCE**: If a specific project rule or requirement is available that declares the color palette and typography for the project, those project-level constraints STRICTLY precede and override the creative aesthetic guidelines of this skill.

The user provides website or frontend requirements: a component, page, application, or interface to build. They may include context about the purpose, audience, or technical constraints.

## Design Thinking

Before coding, understand the context and commit to a BOLD aesthetic direction:
- **Purpose**: What problem does this website solve? Who uses it?
- **Tone**: Pick an extreme: brutally minimal, maximalist chaos, retro-futuristic, organic/natural, luxury/refined, playful/toy-like, editorial/magazine, brutalist/raw, art deco/geometric, soft/pastel, industrial/utilitarian, etc.
- **Constraints**: Technical requirements (framework, performance, accessibility).
- **Differentiation**: What makes this UNFORGETTABLE? What's the one thing someone will remember?

**CRITICAL**: Choose a clear conceptual direction and execute it with precision. Bold maximalism and refined minimalism both work—the key is intentionality, not intensity. You must dynamically determine what aesthetic suits the project best and ensure that the technical implementation goes seamlessly hand-in-hand with the design at hand.

Then implement working code (HTML/CSS/JS, React, Vue, etc.) that is:
- Production-grade and functional.
- Visually striking and memorable.
- **Component-Driven & Modern**: Utilize modern UI libraries and ecosystems like **shadcn/ui**, **21st.dev**, **React Bits**, **Prism**, and other popular tools to construct the website, selecting the ones that best serve the specific design vision.
- **Dimensionally Rich**: Make artifacts into **3D components** (e.g., using Three.js or React Three Fiber) if necessary and if it elevates the chosen aesthetic.
- Cohesive with a clear aesthetic point-of-view.
- Meticulously refined in every detail.

## Frontend Aesthetics Guidelines

Focus on the following (unless overridden by the Rule of Precedence above):
- **Typography**: Choose fonts that are beautiful, unique, and interesting. Avoid generic fonts like Arial and Inter; opt instead for distinctive choices that elevate the frontend's aesthetics. Pair a distinctive display font with a refined body font.
- **Color & Theme**: Commit to a cohesive aesthetic. Use CSS variables for consistency. Dominant colors with sharp accents outperform timid, evenly-distributed palettes.
- **Motion**: Use animations for effects and micro-interactions. Prioritize CSS-only solutions for HTML. Use Framer Motion or similar libraries for React. Focus on high-impact moments: staggered reveals, scroll-triggering, and hover states that surprise.
- **Spatial Composition & Media**: Unexpected layouts. Asymmetry. Overlap. Diagonal flow. Grid-breaking elements. Generous negative space OR controlled density. **When the design requires imagery, use highly relevant, aesthetically matched placeholder images. You MUST include clear code comments next to these images (e.g., ``) so they can be easily swapped later.**
- **Backgrounds & Visual Details**: Create atmosphere and depth. Apply creative forms like gradient meshes, noise textures, geometric patterns, layered transparencies, dramatic shadows, decorative borders, custom cursors, and grain overlays.

NEVER use generic AI-generated aesthetics like overused font families (Inter, Roboto, Arial), cliched color schemes (particularly purple gradients on white backgrounds), or predictable "SaaS-lite" component patterns.

Interpret creatively and make unexpected choices that feel genuinely designed for the context. Vary between light and dark themes, different fonts, and different aesthetics.

**IMPORTANT**: Match implementation complexity to the aesthetic vision. Maximalist designs need elaborate code with extensive animations; minimalist designs need restraint, precision, and careful attention to white space.

Remember: Gemini is capable of extraordinary creative work across multimodal contexts. Don't hold back; show what can truly be created when thinking outside the box and committing fully to a distinctive vision for the web.
